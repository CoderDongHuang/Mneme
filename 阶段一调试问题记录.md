# 阶段一调试问题记录

## 1. Chroma 连接拒绝错误

**错误信息**：`ConnectionRefusedError: [WinError 10061] 由于目标计算机积极拒绝，无法连接`

**原因**：本地测试时 `.env` 配置 `CHROMA_HOST=http://localhost:8000`，代码判断为远程模式，尝试连接不存在的 Chroma 服务器。

**解决方案**：修改 `vector_store.py`，当 `CHROMA_HOST` 包含 `localhost` 或 `127.0.0.1` 时使用 `PersistentClient` 本地持久化模式。

---

## 2. 意图识别返回非法值

**错误信息**：`intent: ingestion` 不在合法意图列表中

**原因**：LLM 返回了不在预定义列表中的意图值。

**解决方案**：在 `nodes.py` 中添加意图白名单校验，非法值自动回退到 `qa`。

---

## 3. Chroma 遥测警告

**错误信息**：`Failed to send telemetry event ClientCreateCollectionEvent: capture() takes 1 positional argument but 3 were given`

**原因**：Chroma 库自身的遥测功能 bug。

**解决方案**：在 `vector_store.py` 开头添加 `os.environ["ANONYMIZED_TELEMETRY"] = "false"` 禁用遥测。

---

## 4. 嵌入模型输入类型错误

**错误信息**：`TypeError: TextInputSequence must be str`

**原因**：PDF 解析后的 chunk 包含 `None` 或非字符串内容，Chroma 默认本地嵌入模型无法处理。

**解决方案**：
- 在 `ingestion.py` 中严格过滤：只保留非空、是字符串、且有实际内容的 chunk
- 强制转换：`str(content).strip()` 确保类型正确

---

## 5. DashScope API Key 无效

**错误信息**：`ValueError: status_code: 401, code: InvalidApiKey, message: Invalid API-key provided`

**原因**：使用了无效的 DashScope API Key（可能是 DeepSeek 的 Key 或已过期）。

**解决方案**：在百炼控制台获取有效的 API Key（`sk-` 开头），更新 `.env` 中的 `DASHSCOPE_API_KEY`。

---

## 6. Chroma 嵌入函数接口不兼容

**错误信息**：`ValueError: Expected EmbeddingFunction.__call__ to have the following signature: odict_keys(['self', 'input']), got odict_keys(['args', 'kwargs'])`

**原因**：Chroma 0.5.3 要求 `EmbeddingFunction` 有明确的参数签名 `(self, input)`，但 LangChain 的 `DashScopeEmbeddings` 不兼容。

**解决方案**：
- 创建 `DashScopeEmbeddingFunction` 包装类实现 Chroma 要求的接口
- `SemanticChunker` 需要 LangChain 的 embeddings，单独创建 `_langchain_embeddings`

---

## 7. 检索结果 metadata 为 None

**错误信息**：`AttributeError: 'NoneType' object has no attribute 'get'`

**原因**：Chroma 返回的 `metadatas` 中包含 `None` 值。

**解决方案**：在 `retriever.py` 中添加空值处理：`results["metadatas"][0][i] if results["metadatas"][0][i] else {}`

---

## 8. Pydantic 验证错误

**错误信息**：`pydantic_core.ValidationError: Input should be a valid string [type=string_type, input_value=None, input_type=NoneType]`

**原因**：`chunk.get("content")` 返回 `None`，Pydantic 要求 `chunk_content` 必须是字符串。

**解决方案**：在 `chat.py` 中跳过内容为空的 chunk。
